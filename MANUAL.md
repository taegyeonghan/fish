# UngdrooFish 사용자 매뉴얼

UngdrooFish는 사용자가 입력한 예측 질문을 바탕으로 관련 로컬 자료를 자동 결합하고, 여러 투자자 페르소나가 토론하는 방식으로 시장 시나리오를 시뮬레이션하는 도구입니다.

이 매뉴얼은 현재 MVP 기준의 사용 흐름을 설명합니다. 사용자는 PDF나 Markdown 파일을 직접 업로드하지 않고, 질문만 입력해 프로젝트를 시작합니다.

## 1. 시작하기

### 접속 주소

개발 환경에서 실행 중이라면 다음 주소로 접속합니다.

- 프론트엔드: `http://localhost:3000`
- 백엔드 상태 확인: `http://localhost:5001/health`

### 필요한 설정

루트의 `.env` 파일에 LLM API 설정이 필요합니다.

```env
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://generativelanguage.googleapis.com/v1beta/openai/
LLM_MODEL_NAME=gemini-3-flash-preview
GEMSTONE_BASE_URL=https://gemstone.ngrok.app
```

### 실행

```bash
npm run dev
```

## 2. 전체 사용 흐름

1. 홈 화면에서 예측 질문을 입력합니다.
2. 시뮬레이션 시간 단위와 예측 기간을 설정합니다.
3. `시뮬레이션 시작하기`를 누릅니다.
4. 시스템이 질문과 관련된 로컬 자료를 자동으로 결합합니다.
5. 온톨로지와 지식그래프가 생성됩니다.
6. 투자자 페르소나를 선택하고 시뮬레이션 환경을 준비합니다.
7. 시뮬레이션을 실행합니다.
8. 결과 리포트를 생성하고 ReportAgent와 대화합니다.

## 3. 질문 입력

홈 화면의 요구사항 입력창에 분석하고 싶은 예측 질문을 자연어로 작성합니다.

좋은 질문은 다음 요소를 포함합니다.

- 분석 대상: 산업, 기업, 자산, 정책, 사건
- 예측 기간: 향후 1개월, 3개월, 6개월 등
- 알고 싶은 결과: 주가 영향, 섹터별 영향, 투자자 반응, 리스크 요인
- 관점: 가치투자, 성장주, 매크로, 리테일 심리 등

예시:

```text
AI 반도체 수요 증가가 엔비디아, TSMC, SK하이닉스에 미칠 영향을 분석하고,
각 투자자 페르소나 관점에서 향후 6개월 투자 시나리오를 제시해줘.
```

```text
중동 지정학 리스크가 유가, 에너지 섹터, 방산주에 미칠 영향을 분석하고,
향후 3개월 동안의 주요 상승/하락 시나리오를 예측해줘.
```

질문만 입력하면 됩니다. 별도의 분석자료 업로드 단계는 없습니다.

## 4. 자동 자료 결합

시뮬레이션을 시작하면 백엔드는 질문을 기준으로 로컬 문서 후보를 검색하고 관련도가 높은 자료를 결합합니다.

현재 MVP에서 사용하는 자료 위치는 다음과 같습니다.

- `Data/`
- `backend/uploads/projects/*/extracted_text.txt`

자동 결합된 자료는 새 프로젝트의 `extracted_text.txt`에 저장되고, 이후 온톨로지 생성과 지식그래프 빌드에 사용됩니다.

주의사항:

- 질문과 관련된 자료가 로컬에 충분히 있어야 더 좋은 그래프가 생성됩니다.
- 자동 수집은 현재 웹 검색이 아니라 로컬 자료 검색입니다.
- 같은 내용의 문서는 중복 제거됩니다.

## 5. 시간 설정

홈 화면에서 시뮬레이션 시간 단위와 예측 기간을 설정합니다.

- 시간 단위: 1라운드가 의미하는 시간 단위입니다.
- 예측 기간: 전체 시뮬레이션이 다루는 기간입니다.

예를 들어 시간 단위를 `일`, 예측 기간을 `14일`로 설정하면, 에이전트들은 약 2주 동안의 시장 변화를 가정하고 상호작용합니다.

## 6. 지식그래프 생성

질문을 제출하면 `프로세스` 화면으로 이동합니다. 이 화면에서는 다음 작업이 순서대로 진행됩니다.

1. 관련 자료 자동 결합
2. 온톨로지 생성
3. 텍스트 청크 분할
4. 엔티티와 관계 추출
5. 로컬 지식그래프 저장

오른쪽 그래프 패널에서 생성 중인 노드와 관계를 확인할 수 있습니다.

그래프 생성이 완료되면 다음 단계로 이동할 수 있습니다.

## 7. 시뮬레이션 환경 설정

지식그래프 생성 후 시뮬레이션 환경을 준비합니다.

주요 작업:

- 질문과 그래프에 맞는 투자자 페르소나 구성
- Reddit/Twitter 형태의 시뮬레이션 환경 설정
- 에이전트별 프로필과 행동 조건 생성
- 시뮬레이션 설정 파일 생성

페르소나는 가치투자자, 성장주 투자자, 모멘텀 트레이더, 매크로 전략가, 퀀트 애널리스트 등 서로 다른 투자 관점을 갖습니다.

## 8. 시뮬레이션 실행

시뮬레이션 실행 화면에서는 다음 정보를 확인할 수 있습니다.

- 현재 라운드
- 에이전트별 게시글과 반응
- 플랫폼별 액션 로그
- 타임라인
- 에이전트 통계

실행 중 문제가 있으면 중지할 수 있습니다. 실행이 끝난 뒤에는 결과 리포트를 생성할 수 있습니다.

## 9. 리포트 생성

리포트 생성 단계에서는 ReportAgent가 시뮬레이션 결과와 지식그래프를 분석해 Markdown 리포트를 작성합니다.

리포트에는 보통 다음 내용이 포함됩니다.

- 핵심 요약
- 주요 시나리오
- 섹터/기업별 영향
- 투자자 페르소나 간 합의점
- 의견이 갈린 쟁점
- 리스크 요인
- 추가로 확인해야 할 근거

리포트 생성 로그와 에이전트 사고 과정을 화면에서 확인할 수 있습니다.

## 10. 결과와 상호작용

리포트가 생성된 뒤에는 ReportAgent와 대화할 수 있습니다.

질문 예시:

```text
가장 강한 상승 시나리오의 핵심 근거만 요약해줘.
```

```text
가치투자자와 성장주 투자자가 의견이 갈린 지점을 설명해줘.
```

```text
방산주 관련 리스크만 따로 정리해줘.
```

## 11. 최근 프로젝트 확인

홈 화면 하단의 최근 시뮬레이션 영역에서 이전 프로젝트를 확인할 수 있습니다.

기존 프로젝트를 열면 다음 정보를 이어서 볼 수 있습니다.

- 생성된 지식그래프
- 시뮬레이션 상태
- 에이전트 프로필
- 결과 리포트

## 12. 문제 해결

### 질문을 입력했는데 진행되지 않음

- 질문 입력창이 비어 있는지 확인합니다.
- 브라우저 콘솔 또는 백엔드 로그를 확인합니다.

### 관련 자료가 부족하다는 결과가 나옴

- `Data/` 폴더에 관련 Markdown 또는 TXT 자료를 추가합니다.
- 이전 프로젝트의 `extracted_text.txt`가 충분히 쌓이면 자동 수집 품질이 좋아집니다.

### 온톨로지 생성 실패

- `.env`의 `LLM_API_KEY`, `LLM_BASE_URL`, `LLM_MODEL_NAME`을 확인합니다.
- LLM API 사용량 제한 또는 네트워크 오류를 확인합니다.

### 시뮬레이션 실행이 멈춤

- 백엔드 로그를 확인합니다.
- 필요하면 시뮬레이션을 중지하고 다시 실행합니다.
- 장시간 실행되는 경우 라운드 수를 줄여 테스트합니다.

### 리포트가 너무 짧거나 부정확함

- 질문을 더 구체적으로 작성합니다.
- 관련 자료를 `Data/`에 보강합니다.
- 시뮬레이션 라운드를 늘립니다.

## 13. 사용 시 주의사항

UngdrooFish의 출력은 투자 조언이 아니라 시뮬레이션 기반 분석 결과입니다.

실제 투자 결정을 내리기 전에 다음을 별도로 확인해야 합니다.

- 최신 공시
- 실제 재무 데이터
- 시장 가격
- 규제 및 정책 변화
- 전문가 검토

## 14. 관리자 참고

질문 기반 자동 자료 결합은 현재 다음 백엔드 모듈에서 처리합니다.

- `backend/app/services/document_collector.py`
- `backend/app/api/graph.py`

자동 수집 범위를 확장하려면 `document_collector.py`에 다음 소스를 추가할 수 있습니다.

- 웹 검색 결과
- 뉴스 API
- 기업 공시
- 리서치 리포트
- 내부 문서 저장소

MVP 기준에서는 로컬 자료 기반으로만 동작합니다.
